# 云电脑连接参数配置文档

## 1. 连接参数概述

云电脑连接参数是用户连接到云电脑实例的关键配置，包括网络地址、认证信息等。正确配置这些参数可以确保用户能够安全、稳定地访问云电脑。

## 2. 核心连接参数

### 2.1 基本连接参数

| 参数名称 | 说明 | 默认值 | 配置方式 |
|---------|------|-------|----------|
| 连接地址 (host) | 云电脑的IP地址和端口 | 动态分配 | UCloud API返回 |
| 端口 (port) | 远程桌面连接端口 | 3389 | 固定值 |
| 用户名 (username) | 登录云电脑的用户名 | Administrator | 固定值 |
| 密码 (password) | 登录云电脑的密码 | 动态生成 | UCloud API返回 |
| UHost ID | 云电脑实例的唯一标识 | 动态分配 | UCloud API返回 |

### 2.2 连接参数详解

#### 2.2.1 连接地址 (host)

- **功能**：云电脑的网络地址，用于建立远程桌面连接
- **获取方式**：从UCloud API的实例信息中获取
- **格式**：IP地址:端口（例如：192.168.1.1:3389）
- **注意事项**：每次启动云电脑时，IP地址可能会变化

#### 2.2.2 端口 (port)

- **功能**：远程桌面协议(RDP)的默认端口
- **默认值**：3389
- **修改方式**：需要在云电脑实例内部修改RDP服务配置
- **注意事项**：修改端口后需要确保防火墙规则相应更新

#### 2.2.3 用户名 (username)

- **功能**：登录云电脑的账户名称
- **默认值**：Administrator
- **修改方式**：在云电脑实例内部创建新用户或修改现有用户
- **注意事项**：默认使用管理员账户，权限较高

#### 2.2.4 密码 (password)

- **功能**：登录云电脑的认证凭据
- **获取方式**：从UCloud API的实例信息中获取（base64解码）
- **安全性**：每次启动云电脑时，密码会重新生成
- **注意事项**：密码仅在本次会话有效，下次启动会自动更改

## 3. 连接参数配置方法

### 3.1 系统默认配置

云电脑连接参数由系统自动配置，无需用户手动设置：

1. **创建云电脑**：系统会自动分配IP地址和生成初始密码
2. **启动云电脑**：系统会更新连接信息，包括新的IP地址和密码
3. **停止云电脑**：连接信息会失效

### 3.2 客户端配置

客户端提供了以下连接参数配置选项：

| 配置项 | 说明 | 默认值 | 配置位置 |
|-------|------|-------|----------|
| 自动连接 | 是否自动启动远程桌面并连接 | true | 客户端配置文件 |
| 自动刷新间隔 | 状态自动刷新的时间间隔（秒） | 5 | 客户端配置文件 |

#### 3.2.1 客户端配置文件

客户端配置文件位于 `client/config.py`，主要配置项包括：

```python
class Config:
    # API配置
    api_base_url: str = "http://localhost:8000"
    
    # 窗口配置
    window_width: int = 800
    window_height: int = 600
    
    # 远程桌面配置
    rdp_auto_connect: bool = True  # 自动连接远程桌面
    
    # 自动刷新间隔（秒）
    auto_refresh_interval: int = 5
    
    # 日志配置
    log_level: str = "INFO"
    log_to_file: bool = True
```

## 4. 连接参数使用方法

### 4.1 通过客户端连接

1. **启动客户端**：运行 `client/main.py` 或打包后的可执行文件
2. **登录系统**：使用账号密码登录
3. **启动云电脑**：点击「启动」按钮
4. **连接远程桌面**：云电脑启动后，点击「连接远程桌面」按钮
5. **自动连接**：客户端会自动处理连接过程

### 4.2 手动连接

如果需要手动连接云电脑，可以按照以下步骤操作：

1. **获取连接信息**：在客户端的「连接信息」区域查看
2. **打开远程桌面客户端**：
   - Windows：搜索「远程桌面连接」
   - macOS：使用 Microsoft Remote Desktop 应用
   - Linux：使用 rdesktop 或 Remmina 等工具
3. **输入连接参数**：
   - 计算机：输入连接地址（例如：192.168.1.1:3389）
   - 用户名：输入 Administrator
   - 密码：输入客户端显示的密码
4. **点击连接**：建立远程桌面连接

## 5. 连接参数管理

### 5.1 连接信息更新

- **启动时更新**：每次启动云电脑时，系统会更新连接信息
- **密码重置**：每次启动时密码会自动重置，提高安全性
- **IP地址变化**：由于使用动态IP，每次启动时IP地址可能会不同

### 5.2 连接信息安全

- **密码保护**：密码仅在云电脑运行期间有效
- **信息隐藏**：密码在客户端中以星号显示，需要点击「复制」按钮查看
- **会话隔离**：每个用户的连接信息相互隔离

## 6. 常见问题与解决方案

### 6.1 连接失败

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 无法连接到远程计算机 | IP地址错误或云电脑未启动 | 检查云电脑状态，确保已启动 |
| 密码错误 | 密码已过期或输入错误 | 重新启动云电脑获取新密码 |
| 连接被拒绝 | 防火墙阻止或端口未开放 | 检查网络连接和防火墙设置 |
| 远程桌面服务未运行 | 云电脑内部服务问题 | 重启云电脑实例 |

### 6.2 连接速度慢

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 网络延迟高 | 网络带宽不足或距离远 | 检查网络连接，选择网络条件好的环境 |
| 图形显示卡顿 | 带宽不足或GPU性能限制 | 降低远程桌面的显示质量 |
| 操作响应慢 | 云电脑资源不足 | 检查云电脑配置，考虑升级资源 |

### 6.3 连接参数配置错误

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 端口错误 | 手动修改了默认端口 | 确认云电脑内部RDP服务端口设置 |
| 用户名错误 | 使用了非默认用户名 | 确认使用 Administrator 用户名 |
| 地址格式错误 | 输入格式不正确 | 确保格式为 IP地址:端口 |

## 7. 技术实现细节

### 7.1 连接参数获取流程

1. **创建云电脑**：调用 UCloud API 创建实例，获取初始连接信息
2. **启动云电脑**：调用 UCloud API 启动实例，获取最新连接信息
3. **更新连接信息**：将连接信息存储到数据库
4. **客户端获取**：客户端通过 API 获取连接信息

### 7.2 连接信息存储

连接信息存储在数据库的 `container` 表中：

- `connection_host`：连接地址
- `connection_port`：连接端口
- `connection_username`：连接用户名
- `connection_password`：连接密码

### 7.3 安全措施

- **密码加密**：密码在传输和存储过程中进行加密
- **会话隔离**：每个用户只能访问自己的连接信息
- **定期更新**：每次启动时自动更新密码
- **权限控制**：只有云电脑所有者可以查看连接信息

## 8. 最佳实践

### 8.1 连接参数管理建议

1. **使用客户端自动连接**：客户端提供一键连接功能，简化连接过程
2. **保存连接信息**：在云电脑运行期间，可以保存连接信息以便快速重连
3. **定期更新密码**：虽然系统会自动更新密码，但建议用户在必要时手动修改
4. **使用强密码**：如果手动设置密码，应使用强密码策略

### 8.2 网络配置建议

1. **确保网络稳定**：使用稳定的网络连接，避免频繁断连
2. **适当带宽**：建议使用至少10Mbps的网络带宽
3. **防火墙设置**：确保本地防火墙允许RDP连接（端口3389）
4. **VPN使用**：对于敏感操作，建议使用VPN连接

### 8.3 安全使用建议

1. **不要共享密码**：连接密码属于个人敏感信息，不要与他人共享
2. **及时退出**：使用完毕后及时退出远程桌面连接
3. **关闭不必要的服务**：在云电脑中关闭不必要的网络服务
4. **定期更新系统**：保持云电脑操作系统的安全更新

## 9. 故障排查

### 9.1 连接问题排查步骤

1. **检查云电脑状态**：确保云电脑处于运行状态
2. **验证连接信息**：确认IP地址、端口、用户名和密码正确
3. **测试网络连接**：使用ping命令测试网络连通性
4. **检查防火墙设置**：确保本地和云电脑的防火墙允许RDP连接
5. **重启云电脑**：如果以上步骤都无效，尝试重启云电脑

### 9.2 常见错误代码及解决方案

| 错误代码 | 错误信息 | 解决方案 |
|---------|---------|----------|
| 0x204 | 无法连接到远程计算机 | 检查网络连接和云电脑状态 |
| 0x112f | 远程会话被中断 | 检查网络稳定性，重新连接 |
| 0x104 | 无法连接到远程服务器 | 检查云电脑是否已启动 |
| 0x3 | 远程桌面服务不可用 | 重启云电脑实例 |
