# 云电脑参数配置文档

## 1. 配置概述

云电脑参数配置是管理云电脑运行成本和使用限制的重要功能。通过合理配置参数，可以确保系统的可持续运营，同时为用户提供公平、透明的使用体验。

## 2. 配置参数说明

### 2.1 核心配置参数

| 参数名称 | 说明 | 默认值 | 最小值 | 操作权限 |
|---------|------|-------|-------|----------|
| 每分钟价格 (price_per_minute) | 云电脑运行每分钟的费用 | 0.5元 | 0.01元 | 超级管理员 |
| 启动最低余额 (min_balance_to_start) | 启动云电脑所需的最低账户余额 | 2.5元 | 0元 | 系统默认 |
| 自动停止阈值 (auto_stop_threshold) | 当余额低于此值时自动停止云电脑 | 0元 | 0元 | 系统默认 |

### 2.2 配置参数详解

#### 2.2.1 每分钟价格

- **功能**：决定云电脑运行时的计费标准
- **影响范围**：仅影响新创建的云电脑实例，已创建的实例保持创建时的价格（价格快照机制）
- **设置建议**：根据硬件成本、运营成本和市场需求合理设置
- **调整频率**：建议在业务初期或成本变化时调整

#### 2.2.2 启动最低余额

- **功能**：防止用户账户余额过低时频繁启停云电脑
- **计算方式**：默认设置为5分钟的费用（0.5元/分钟 × 5分钟 = 2.5元）
- **影响范围**：影响所有用户启动云电脑的操作
- **设置建议**：设置为3-5分钟的费用，平衡用户体验和系统稳定性

#### 2.2.3 自动停止阈值

- **功能**：当用户余额低于此值时，系统会自动停止云电脑
- **默认值**：0元（表示余额为0时才自动停止）
- **设置建议**：可根据业务需求调整，如设置为1元以提前预警

## 3. 配置操作流程

### 3.1 前端配置流程

1. **登录系统**：使用管理员账号登录后台管理系统
2. **进入配置页面**：点击左侧菜单栏的「系统配置」
3. **调整参数**：
   - 在「每分钟价格」输入框中输入新的价格
   - 系统会自动计算并显示建议的最低启动余额
4. **保存配置**：点击「保存配置」按钮
5. **确认成功**：系统会显示「配置保存成功」的提示

## 4. 配置注意事项

### 4.1 价格调整注意事项

1. **价格快照机制**：价格调整只影响新创建的云电脑实例，已创建的实例保持创建时的价格
2. **日志记录**：所有价格变动都会被记录在系统日志中，便于审计
3. **权限控制**：只有超级管理员可以调整价格参数
4. **余额检查**：价格调整后，系统会在用户启动云电脑时检查余额是否满足最低要求

### 4.2 业务影响

1. **用户体验**：合理的价格设置可以提高用户满意度
2. **系统负载**：过低的价格可能导致系统负载过高
3. **运营成本**：价格设置应覆盖硬件和运营成本
4. **市场竞争力**：价格应参考市场同类产品的定价

## 5. 常见问题解答

### 5.1 价格调整后，已创建的云电脑会涨价吗？

不会。系统采用价格快照机制，云电脑创建时会记录当时的价格，后续价格调整不会影响已创建的实例。

### 5.2 最低启动余额可以自定义吗？

目前最低启动余额是系统根据每分钟价格自动计算的（默认5分钟费用），不支持手动调整。

### 5.3 如何查看历史价格调整记录？

所有价格调整都会被记录在系统日志中，管理员可以通过「系统日志」页面查看详细的操作记录。

### 5.4 价格设置过低会有什么影响？

价格设置过低可能导致：
- 系统负载过高，影响服务质量
- 运营成本无法覆盖，影响系统可持续性
- 用户可能过度使用资源，导致资源浪费

### 5.5 如何确定合理的价格？

建议考虑以下因素：
- 硬件成本（GPU、CPU、内存等）
- 带宽和存储成本
- 运营维护成本
- 市场同类产品的定价
- 目标用户群体的消费能力

## 6. 技术实现细节

### 6.1 后端配置管理

配置参数存储在数据库中，通过 `config_service` 进行管理：

- 获取配置：`config_service.get_all_configs()`
- 设置配置：`config_service.set_config()`
- 获取价格：`config_service.get_price_per_minute()`
- 获取最低余额：`config_service.get_min_balance_to_start()`

### 6.2 前端配置界面

前端使用 Vue 3 + Element Plus 实现，通过 API 与后端交互：

- 获取配置：`getConfig()`
- 更新价格：`updatePrice()`

### 6.3 价格快照机制

当用户创建云电脑时，系统会记录当时的价格，后续价格调整不会影响该实例：

```python
# 创建容器时记录价格
price_per_minute = await config_service.get_price_per_minute(db)
container = await container_service.create(
    db,
    current_user.id,
    {...},
    price_per_minute,  # 记录当前价格
)
```

## 7. 配置建议

### 7.1 初始配置建议

- **每分钟价格**：0.5-1.0元（根据硬件配置调整）
- **最低启动余额**：默认值（5分钟费用）
- **自动停止阈值**：0元

### 7.2 运营策略建议

1. **促销活动**：可临时降低价格以吸引新用户
2. **阶梯定价**：根据使用时长设置不同的价格档位
3. **会员制度**：为长期用户提供折扣
4. **监控调整**：定期监控系统负载和用户反馈，及时调整价格

## 8. 故障排查

### 8.1 配置保存失败

- **原因**：可能是权限不足或网络问题
- **解决方法**：确保使用超级管理员账号登录，检查网络连接

### 8.2 价格调整后不生效

- **原因**：价格调整只影响新创建的实例
- **解决方法**：删除旧实例并创建新实例以应用新价格

### 8.3 启动时余额检查失败

- **原因**：用户余额低于最低启动要求
- **解决方法**：提醒用户充值，或调整最低启动余额设置

## 9. 总结

云电脑参数配置是系统运营的重要组成部分，合理的配置可以平衡用户体验和系统可持续性。通过本文档的指导，管理员可以根据业务需求灵活调整配置参数，确保系统的稳定运行和良好的用户体验。
